# 10.8.9 Unit Attention Condition

Unit Attention Condition (UAC) is a condition which needs to be serviced before the logical unit can process commands.

Table 10.66 shows all events which shall establish UAC. Power-on, HW Reset, EndPointReset, and Host UniProWarm Reset shall establish UAC on all LUs including all Well-known LUs (REPORT LUNS, UFS Device, RPMB and BOOT). LU Reset shall establish UAC only for an addressed LU.

**Table 10.66 — Events for UAC Establishment**

| Event | Value |
|-------|--------|
| Power-on | |
| HW Reset | All LUs including all well-known LUs |
| EndPointReset | |
| Host UniPro Warm Reset | |
| LU Reset | Addressed LU |

UAC on each LU shall be cleared before it can be accessed successfully (See Table 10.67 for exceptions). Except for some specific commands listed in Table 10.67, if a command is sent to a LU with a pending UAC, the command will fail. The device server shall respond with CHECK CONDITION status, Sense Key set to UNIT ATTENTION. Afterwards the device server shall clear UAC on the LU. Table 10.67 shows some exceptional commands on UAC behavior.

**Table 10.67 — Commands for Exceptional Behavior on UAC**

| Command | Status on pending UAC | | UAC after command response |
|---------|----------------------|---|----------------------------|
| | | | |
| REQUEST SENSE | GOOD (1) | - | Clear |
| INQUIRY | GOOD | - | Not clear |
| REPORT LUNS | GOOD | - | Not clear |
| Others | CHECK CONDITION | UNIT ATTENTION | Clear |

**NOTE 1** If the REQUEST SENSE command was received with a pending unit attention condition, the returned sense data will indicate the cause of the unit attention condition (See 11.3.17.4)
