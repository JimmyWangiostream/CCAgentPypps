### 10.7.13 Data Out Transfer Rules (cont'd)

[THIS IS FIGURE: A sequence diagram showing data transfer between Host and Device. The diagram illustrates:

- Host sends COMMAND UPIU(Write) with Expected Data Transfer Length = 4MB
- Device responds with RTT UPIU¹ containing Data Buffer Offset = 00000h, Data Transfer Count = 48KB
- Host sends DATA OUT UPIU¹ with Data Buffer Offset = 00000h, Data Transfer Count = 48KB
- Device responds with RTT UPIU² containing Data Buffer Offset = 0C000h, Data Transfer Count = 32KB
- Host sends DATA OUT UPIU² with Data Buffer Offset = 0C000h, Data Transfer Count = 32KB
- Device responds with RTT UPIU³ containing Data Buffer Offset = 14000h, Data Transfer Count = 16KB
- Host sends DATA OUT UPIU³ with Data Buffer Offset = 14000h, Data Transfer Count = 16KB]

**Figure 10.9 — Example for Data Out Transfer Rule 2**

• **Rule 3** – Across all logical units, DATA OUT UPIUs shall be sent in the same order of RTT UPIUs; data for RTT^N shall be transferred before transferring data for RTT^N+1.
  ○ For example, if the host first receives RTT UPIU¹ and then RTT UPIU² from the device, it shall send data related to the first request (Data1) prior to data related to the second request (Data2).
